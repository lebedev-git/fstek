# MCP-сервер трекера ФСТЭК №117

Даёт Claude (или другому MCP-клиенту) инструменты для анализа проектов на соответствие Приказу №117.
Пишет в тот же `src/data/assessments.json`, что и веб-дашборд → оценки видны в UI сразу.

## Петля работы

```
Дал проект → Claude читает код (своими файловыми инструментами)
           → сверяет с мерами (list_measures)
           → создаёт/находит карточку (create_project / list_projects)
           → шлёт оценки (submit_assessment)
           → дашборд платформы показывает балл и пробелы
```

## Протокол для агента (handshake)

При подключении клиент получает `instructions` — краткий порядок работы. Полный протокол
(шаги, семантика статусов, правила «не завышать pass») агент берёт двумя способами:

- инструмент **`get_protocol`** — гарантированно, через tool-discovery;
- ресурс **`protocol://fstek-117`** — для клиентов, читающих resources.

Порядок: `list_measures` (сигнал «подключился») → `create_project` (базовые поля) →
`scan_project` → разбор `manual`/`fail` через `set_assessment`/`submit_assessment` → `get_project`.

## Инструменты

| Tool | Назначение |
|------|-----------|
| `get_protocol` | протокол работы аудитора (прочитать первым) |
| `list_measures` | каталог мер №117 (чек-лист для анализа) |
| `list_projects` | карточки продуктов с баллами |
| `get_project` | детали продукта + текущие оценки |
| `create_project` | завести карточку продукта |
| `update_project` | изменить метаданные/статус (draft/in_review/attested) |
| `set_assessment` | оценить одну меру |
| `submit_assessment` | пакетно проставить оценки после анализа (главный) |
| `scan_project` | детерминированный автоскан кода по пути карточки |

## Запуск вручную

```bash
node mcp/server.mjs
```

Общается по stdio (JSON-RPC). Логи — в stderr.

## Подключение к Claude Code

Добавить в конфиг MCP-серверов (`.mcp.json` в проекте или пользовательский конфиг):

```json
{
  "mcpServers": {
    "fstek-117": {
      "command": "node",
      "args": ["C:\\Disk D\\Project\\fstek\\platform\\mcp\\server.mjs"]
    }
  }
}
```

После подключения Claude видит инструменты `fstek-117:*`. Типовой сценарий:
дать путь к проекту → «проанализируй на соответствие №117 и занеси в трекер» →
Claude читает код, вызывает `create_project` + `submit_assessment` → открываешь дашборд.

## Требования

- Node ≥ 18 (ESM, глобальный fetch).
- Зависимости ставятся общим `npm install` в `platform/`.

## Модель развёртывания (сервер)

Целевая схема — **доверенный stdio на сервере**:

- Агент получает доступ к серверу, подключается к MCP по stdio (как локально). Сеть наружу не открывается.
- **Аутентификации внутри приложения нет:** доступ к серверу = доступ к трекеру. Ставить только
  под одного доверенного оператора/агента.
- **Хранилище общее:** один `assessments.json`, все проекты видны всем. Изоляции по клиентам нет.
- `scan_project` с `overridePath` читает любой путь на сервере — под доверенной моделью это ок
  (у агента и так шелл). Наружу MCP/веб **не выставлять** без auth + песочницы путей.

## Сетевой доступ (HTTP) — менять статусы с любого компьютера

Помимо stdio есть **сетевой транспорт**: агент Claude с любого компьютера обращается по HTTP
к MCP-серверу, живущему рядом с трекером и данными.

```
Claude (любой комп) ──HTTP + Bearer-токен──► http-server.mjs → core.mjs → assessments.json → дашборд
```

**Запуск (на сервере/локально):**

1. Создать `platform/.env` (в git не идёт):
   ```
   MCP_TOKEN=<сильный токен, напр. `node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"`>
   MCP_HTTP_PORT=1338
   MCP_HTTP_HOST=0.0.0.0
   ```
2. `cd platform && npm run mcp:http` → слушает `http://<host>:1338/mcp`.
3. Health-check: `GET http://<host>:1338/` → `ok` (без токена).

**Подключение клиента** (`.mcp.json` или `--add-mcp`):
```json
{ "type": "http", "url": "http://<сервер>:1338/mcp",
  "headers": { "Authorization": "Bearer <MCP_TOKEN>" } }
```

**Транспорт:** Streamable HTTP, stateful-сессии (`mcp-session-id`). POST — вызовы, GET — SSE-поток,
DELETE — закрытие сессии. Первый POST должен быть `initialize`.

**Безопасность (обязательно):**
- **Токен обязателен** — без `MCP_TOKEN` сервер не стартует. Запрос к `/mcp` без `Bearer` → **401**.
- **TLS** для интернета — только через обратный HTTPS-прокси (caddy/nginx). Голый HTTP — лишь
  доверенная LAN/localhost.
- `MCP_HTTP_HOST=0.0.0.0` открывает порт наружу — за пределами доверенной сети закрыть фаерволом/прокси.
- Запись в JSON атомарна+сериализована (ядро) — параллельные сетевые записи файл не побьют.
- Держать токен в секрете; при утечке — сменить в `.env` и перезапустить.

Мультиарендность/изоляция данных по клиентам — следующая фаза (сейчас склад общий).

## Граница

MCP пишет оценки, но **сам анализ кода** делает агент (Claude) своими средствами чтения файлов
и `scan_project`. Сервер = мост между выводами агента и хранилищем платформы.
Балл = трекинг-готовность, не гарантия аттестации.
